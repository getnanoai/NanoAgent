Enterprise Gateway

Govern every model call across your organization

NanoAgent Gateway is the control plane for enterprise AI. Put a single OpenAI-compatible endpoint in front of every provider, and give security, finance, and platform teams what they need: policy enforcement, guardrails, cost attribution, SSO, and an immutable audit trail — self-hosted in your own cloud. Drop-in OpenAI-compatible proxy

Change one base URL. The Gateway speaks the OpenAI API your tools already use.

import { streamText } from 'ai'import { createOpenAI } from '@ai-sdk/openai' const nano = createOpenAI({  baseURL: 'https://app.getnanoai.com/v1',  apiKey: process.env.NANOAGENT_API_KEY,}) const result = streamText({  model: nano.chat('anthropic/claude-opus-4-8'),  prompt: 'Why is the sky blue?',})
import osfrom openai import OpenAI client = OpenAI(    base_url="https://app.getnanoai.com/v1",    api_key=os.environ["NANOAGENT_API_KEY"],) stream = client.chat.completions.create(    model="anthropic/claude-opus-4-8",    messages=[{"role": "user", "content": "Why is the sky blue?"}],    stream=True,)
# point any OpenAI-compatible client at the Gatewaycurl https://app.getnanoai.com/v1/chat/completions \  -H "Authorization: Bearer $NANOAGENT_API_KEY" \  -H "Content-Type: application/json" \  -d '{    "model": "anthropic/claude-opus-4-8",    "messages": [{"role": "user", "content": "Why is the sky blue?"}],    "stream": true  }'

Standard routes: POST /v1/chat/completions and GET /v1/models. Streaming responses, headers, and finish reasons pass through transparently.

Why Nano Gateway?

One control plane between your apps and every model — without rewriting a line of client code.

🔌

Zero migration

Keep your existing OpenAI-compatible SDKs and prompts. Swap one base URL and you're routed through the Gateway.

🛡️

Governance built in

Policy enforcement, guardrails, SSO, and an immutable audit trail wrap every call — no extra integration work.

💸

Cost in your control

Attribute spend per team, override pricing per model, and meter usage for billing across every provider.

🏢

Self-hosted

Runs in your own cloud. Keys, prompts, and traffic never leave your perimeter.

🌐

Model routing & providers

One catalog across every upstream provider, with central control over what ships to users.

🔀

Multi-provider proxy

Route to any provider exposing an OpenAI-compatible chat-completions API, behind a single team endpoint.

🗝️

Central credential vault

Store and rotate upstream provider keys server-side — clients authenticate to the Gateway, never to the provider.

🎚️

Model enable / disable

Selectively expose upstream models to end users and refresh model catalogs on demand.

🏷️

Per-model pricing

Override pricing at model granularity — input, output, cached-input, and cache-write rates.

🪄

Transparent model rewrite

Map a client's requested model ID to the real upstream name, with optional per-model system-prompt injection.

📏

Context-window sync

Context windows are synchronised from upstream on each provider refresh and tracked per model.

⚖️

Policy engine

Scope-based access control over who can call which model, how much, and from where.

🎯

Scoped rules

Target rules at the whole org, a team, an API-key scope, or an individual user.

🚦

Allow / deny modes

Run default-allow with a blocklist or default-deny with a whitelist, depending on your risk posture.

💰

Budget limits

Per-scope USD spend ceilings stop runaway cost before the request goes upstream.

⏱️

Token & request limits

Windowed token and request-count caps per scope, over an hour, day, or month.

🎲

Sampling enforcement

Apply a rule probabilistically — e.g. audit or shadow-check 10% of calls.

🌍

IP / CIDR rules

Allow or deny by client IP or network range, with trusted-proxy resolution behind load balancers.

🛡️

Guardrails

Inline content checks that run before a single response byte reaches the client.

💉

Prompt-injection detection

Pattern-based detection against OWASP LLM injection techniques.

🔏

Sensitive-data detection

Catch PII, API keys, and credential patterns in prompts and responses.

✂️

Block or redact

Per-pattern action modes — block the request outright or mask the matched content.

⚡

Real-time evaluation

Guardrails run inline, so a block happens before any bytes are streamed back.

📊

🪙

Granular metering

Track input, output, and cached tokens, request counts, and latency on every call.

🔬

Multi-dimensional slicing

Break usage down by provider, model, team, user, API key, app, and project.

📈

Logs & summaries

Query per-request logs and aggregated trends over windows from 1 to 365 days.

🧾

Live quota usage

See live consumption against plan quotas, separate from audit-gated reporting.

💳

Stripe billing

Checkout, subscription management, and invoice retrieval with normalised webhook events.

🛑

Quota enforcement

RBAC

Audit & compliance

Alerts & incidents

Metric-driven alerting with on-call workflows built in.

Alert rules

Incident grouping

Silencing & maintenance
